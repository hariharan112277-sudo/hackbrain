# Relational & Logical Entity Storage Interface Mapping (database_mapping.md)

This section details the mappings to the Member 2 structural database components. These schemas and layouts are completely owned and operated by Member 2; our backend contracts map explicitly onto these boundaries without modifying them.

## 1. Core Target Tables & Structural Relationships Index

### 1.1 `iob_metadata_machines` (Relational Operational Store)

- **Purpose**: Manages structural records for functional digital twin industrial asset tracking nodes.
- **Primary Key**: `uuid` with UUIDv4 compliance tracking format.
- **Indexes Built**: `idx_machines_operational_status` (B-Tree), `idx_machines_manufacturer` (B-Tree).
- **Interface Layer Resolution**: `MachineRepository.get_by_id(uuid: UUID) -> Optional[MachineEntity]`.

### 1.2 `iob_telemetry_timeseries` (Hyper-Scale Partitioned Store)

- **Purpose**: Stores high-frequency metric measurements generated by active sensor network nodes.
- **Primary Key Combination**: `(sensor_uuid, timestamp DESC)`.
- **Data Partition Strategy**: Weekly time-series range boundary slicing partitions.
- **Interface Layer Resolution**: `TelemetryRepository.query_range(sensor_id: UUID, start: datetime, end: datetime) -> list[TelemetryRecord]`.

### 1.3 `iob_runtime_alarms` (Runtime State Store)

- **Purpose**: Stores active and recent alarm state transitions for operators and dashboards.
- **Primary Key**: `alarm_uuid`.
- **Index Expectations**: `idx_alarm_state`, `idx_alarm_machine_uuid`, `idx_alarm_trigger_timestamp`.
- **Interface Layer Resolution**: `AlarmRepository.find_by_status(state: str) -> list[AlarmRecord]`.

## 2. Layer Transformation Direct Mapping Reference Matrix

| API Endpoint URI Target | Primary Target Storage Table | Target Structural Relationship Fields | Required Invocation Method |
| :--- | :--- | :--- | :--- |
| `/api/v1/machines` | `iob_metadata_machines` | None (Direct Table Read) | `MachineRepository.list_all()` |
| `/api/v1/historical/query` | `iob_telemetry_timeseries` | `sensor_uuid` foreign key join | `TelemetryRepository.query_range()` |
| `/api/v1/alarms/active` | `iob_runtime_alarms` | Filter condition where `state == 'ACTIVE'` | `AlarmRepository.find_by_status()` |

## 3. Contract Projection Rules

- Storage entities map to `integration.contracts` internal DTOs before being projected into `integration.backend_contracts` response models.
- Driver-specific columns, partition metadata, internal revision tokens, and ORM session state must never cross the API boundary.
- Dynamic metadata fields must be structurally validated during Service-layer DTO mapping before being included in API response payloads.
